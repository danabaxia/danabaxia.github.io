I build the systems that decide whether a battery ships or gets pulled off the line.

At Enphase Energy, I design and maintain the manufacturing test infrastructure for the IQ Battery R4 production line. My work spans the full stack of production testing: from Excel-based test requirement documents and Python test frameworks, through CAN bus hardware communication, to real-time yield analysis with SQL.

Recently, I've been pioneering the use of AI agents as development partners in this space — building validation systems where Claude Code and an engineer collaborate to validate test changes without requiring physical hardware or lab time.

What I Do

Manufacturing Test Frameworks

Python-based systems that orchestrate multi-step test sequences, communicate with battery hardware over CAN bus, and determine pass/fail decisions on the production line. Every unit tested, every result traceable.

AI-Assisted Validation

Built a three-mode validation system where an AI agent and engineer validate manufacturing test changes together — 124 automated checks, zero hardware required. The AI writes code, runs tests, and iterates on failures in the same session.

Virtual Test Runners

Simulation layers that replicate real hardware behavior from golden unit profiles. 20 hardware-bound methods intercepted, 5-state battery state machine with validated transitions, per-device tracking — full test validation on a laptop.

Production Data Analysis

SQL-driven yield analysis, failure Pareto charts, and root cause investigation across multi-site test stations. Turning raw production data into actionable insights — identifying whether a failure is a unit issue, a fixture issue, or a software issue.

What Drives Me

Manufacturing test engineering sits at a unique intersection: hardware domain knowledge, software engineering, and production constraints (throughput, reliability, cost). Most of this work happens behind closed doors and never gets written about.

I believe the next leap in manufacturing test efficiency comes from AI-assisted development — not replacing engineers, but giving them a partner that can write code, run 94 checks in 30 seconds, and iterate on failures without context switching. I'm building that workflow and writing about it here.

I'm looking for opportunities where I can apply this combination of hardware-aware software engineering, test automation at scale, and AI-augmented workflows to solve hard problems in production, manufacturing, or embedded systems.
